## 什么是量化交易策略

量化交易策略是一套用数学、统计和编程语言精确定义的交易规则集合。与主观交易不同，量化策略不依赖交易者的直觉或经验判断，而是通过严谨的算法来决定何时买入、何时卖出、买卖多少。

一个完整的量化交易策略至少包含以下三个核心要素：

1. **入场规则**：定义何时建立仓位。例如"当5日均线上穿20日均线时买入"。
2. **出场规则**：定义何时平仓。例如"当5日均线下穿20日均线时卖出"或"持有5个交易日后平仓"。
3. **仓位管理规则**：定义每次交易投入多少资金。例如"每次交易使用总资金的10%"。

### 策略、信号与执行的关系

理解"信号"与"执行"的区别对于构建稳健的量化策略至关重要：

| 概念 | 定义 | 示例 |
|------|------|------|
| **信号 (Signal)** | 一种输入信息，指示当前市场状态或预期方向 | RSI < 30 表示超卖 |
| **策略 (Strategy)** | 将信号转化为具体交易动作的完整规则 | RSI < 30 时买入，RSI > 70 时卖出 |
| **执行 (Execution)** | 将策略生成的交易指令实施到市场中 | 以限价单买入100股 |

> **重要概念**：信号不等于策略。一个策略可能综合多个信号（如动量信号 + 波动率信号），通过加权或逻辑组合生成最终的投资决策。执行层面则关注交易成本、滑点、市场冲击等实际因素。

### 策略的生命周期

任何一个量化策略都会经历以下完整的生命周期：

```
[研究设计] → [回测验证] → [模拟交易] → [实盘上线] → [监控迭代]
```

#### 阶段一：研究设计

在这一阶段，量化研究员基于金融理论或数据观察提出假设，并设计策略逻辑。这通常包括：

- **数据收集**：获取历史行情数据（日线、分钟线、tick数据）、基本面数据、另类数据等。
- **特征工程**：构建预测信号或因子，如计算移动平均、RSI、布林带等。
- **规则定义**：明确入出场条件、止损止盈规则、仓位计算公式。

```python
# 策略研究阶段：生成简单的均线交叉信号
import pandas as pd
import numpy as np

def generate_signal(df, short_window=5, long_window=20):
    """
    生成均线交叉信号
    1 = 买入信号, -1 = 卖出信号, 0 = 无信号
    """
    df = df.copy()
    df['ma_short'] = df['close'].rolling(short_window).mean()
    df['ma_long'] = df['close'].rolling(long_window).mean()
    
    # 金叉：短线上穿长线
    df['signal'] = 0
    df.loc[df['ma_short'] > df['ma_long'], 'position'] = 1
    df.loc[df['ma_short'] <= df['ma_long'], 'position'] = -1
    
    # 信号只在交叉时刻产生
    df['signal'] = df['position'].diff().fillna(0)
    df.loc[df['signal'] > 0, 'signal'] = 1   # 买入
    df.loc[df['signal'] < 0, 'signal'] = -1  # 卖出
    
    return df
```

#### 阶段二：回测验证

回测（Backtest）是将策略应用于历史数据，模拟其历史表现的过程。核心评估指标包括：

| 指标 | 公式/含义 | 说明 |
|------|-----------|------|
| 年化收益率 | (最终净值)^(252/n) - 1 | 年化后的平均收益 |
| 最大回撤 | max((峰值-谷值)/峰值) | 策略最大亏损幅度 |
| 夏普比率 | (年化收益 - 无风险利率) / 年化波动率 | 风险调整后收益 |
| 胜率 | 盈利交易数 / 总交易数 | 交易成功率 |
| 盈亏比 | 平均盈利 / 平均亏损 | 单笔盈亏大小比 |

> **回测陷阱**：初学者常犯的错误包括——前视偏差（使用未来数据）、生存者偏差（只使用还存活的股票）、过拟合（参数调得太"完美"但在实盘中失效）。务必使用样本外数据验证策略的稳健性。

#### 阶段三：模拟交易

模拟交易（Paper Trading）是在真实市场环境中使用虚拟资金运行策略。这一阶段帮助验证：

- 策略逻辑在实时数据中的表现是否与回测一致
- 下单执行过程是否有逻辑缺陷
- 延迟、数据缺失等实际问题的处理能力

#### 阶段四：实盘上线

实盘交易将策略投入真实资金。此时需要关注：

- 交易成本（佣金、印花税、滑点）
- 资金管理和风险控制
- 与券商的API对接

#### 阶段五：监控迭代

策略上线后需要持续监控其表现。当策略出现以下情况时需要考虑调整或下线：

- 回撤超过预设阈值
- 市场制度发生重大变化
- 因子有效性出现明显衰减

### 量化策略的分类

量化策略可以从多个维度进行分类：

**按持仓周期分类：**

| 类型 | 持仓时间 | 交易频率 | 典型策略 |
|------|----------|----------|----------|
| 高频交易 | 毫秒到分钟 | 极高 | 做市、套利 |
| 日内交易 | 分钟到小时 | 高 | 趋势跟踪、反转 |
| 短线交易 | 1-5天 | 中 | 事件驱动、动量 |
| 中线交易 | 1-4周 | 中低 | 因子轮动、均值回归 |
| 长线交易 | 月到年 | 低 | 价值投资、资产配置 |

**按策略逻辑分类：**

| 大类 | 子类 | 核心逻辑 |
|------|------|----------|
| 趋势跟踪 | 均线交叉、通道突破 | 顺势而为，截断亏损让利润奔跑 |
| 均值回归 | 配对交易、统计套利 | 价格偏离均值后会回归 |
| 因子投资 | 多因子选股、因子轮动 | 通过多个因子综合打分选股 |
| 事件驱动 | 财报套利、并购套利 | 利用特定事件带来的价格异动 |
| 统计套利 | 协整套利、波动率套利 | 利用资产间的统计关系获利 |

### 策略开发的最佳实践

1. **保持简单**：复杂的策略不一定更好。简单且逻辑清晰的策略更容易理解和维护，也往往更稳健。
2. **避免过度优化**：不要在回测中不断调整参数使曲线完美，这会导致样本内过拟合。
3. **考虑交易成本**：忽略手续费和滑点的回测结果毫无意义。
4. **分散化**：单一策略风险较大，组合多个低相关性的策略可以平滑收益曲线。
5. **持续学习**：市场在变化，策略需要不断迭代和优化。
