## 综合案例项目概述

### 1.1 项目目标

本章将带领你完成一个完整的端到端（End-to-End）量化交易策略开发项目。我们将从零开始，使用Python和DolphinDB两种工具，构建一个具有止损保护和风险控制功能的趋势跟踪策略，并在历史数据上进行完整的回测验证。

通过本项目，你将完整经历量化交易策略的标准开发流程：

```
数据获取 → 数据清洗 → 信号生成 → 仓位管理 → 回测执行 → 绩效评估
```

> **学习目标**：完成本章后，你将能够独立完成一个完整的量化策略，从数据到评估的全流程。这是从"理论学习者"到"实践者"的关键一步。

### 1.2 项目架构

本项目采用Python + DolphinDB双引擎架构：

```
┌─────────────────────────────────────────────────┐
│                   整体架构                        │
├─────────────────────────────────────────────────┤
│  Python层：数据获取、可视化、策略探索              │
│  ├── 数据获取（tushare / akshare）               │
│  ├── 数据清洗与预处理                             │
│  ├── 信号生成（pandas + numpy）                  │
│  ├── 回测引擎（自建向量化回测）                    │
│  └── 结果可视化（matplotlib）                     │
├─────────────────────────────────────────────────┤
│  DolphinDB层：高性能计算、生产环境回测              │
│  ├── 数据存储（分布式数据库）                      │
│  ├── 因子计算（TA模块）                           │
│  ├── 批量回测（回测引擎）                          │
│  └── 绩效分析报告                                 │
├─────────────────────────────────────────────────┤
│  协作方式：DolphinDB Python API 桥接              │
└─────────────────────────────────────────────────┘
```

分工原则：
- **Python**：负责探索性分析、结果可视化和策略的快速原型验证
- **DolphinDB**：负责大规模数据存储、高性能因子计算和生产级回测

### 1.3 策略概述

我们将开发一个经典的**双均线趋势跟踪策略**，具体构成：

| 组件 | 实现方式 | 参数 |
|------|----------|------|
| **趋势信号** | MA20与MA60的金叉/死叉 | 快线20日，慢线60日 |
| **仓位管理** | ATR动态仓位 | ATR倍数 = 2 |
| **风险控制** | 移动止损（Trailing Stop） | 回撤比例 = 5% |
| **交易频率** | 日频 | 每日收盘后判断 |

**策略信号规则**：

- **买入信号**：MA20上穿MA60（黄金交叉），且当前无持仓
- **卖出信号**：MA20下穿MA60（死亡交叉），或触发移动止损
- **仓位大小**：固定股数或基于ATR动态计算

### 1.4 数据范围

本案例将使用以下数据和参数：

| 项目 | 设定 |
|------|------|
| 标的 | 沪深300成分股（示例：贵州茅台 600519.SH） |
| 数据区间 | 2015年1月 - 2023年12月 |
| 数据频率 | 日线（OHLCV） |
| 初始资金 | 100,000元 |
| 手续费 | 万分之三（0.03%） |
| 滑点 | 万分之一（0.01%） |

### 1.7 项目里程碑

整个项目分为四个阶段，对应本章的四个小节：

| 阶段 | 内容 | 产出物 | 预计耗时 |
|------|------|--------|:---:|
| 阶段一 | 数据准备 | 清洗后的OHLCV数据 | 30分钟 |
| 阶段二 | 策略开发 | 信号生成和仓位管理代码 | 45分钟 |
| 阶段三 | 回测实现 | 完整回测代码和绩效指标 | 45分钟 |
| 阶段四 | 优化总结 | 策略评估和优化建议 | 30分钟 |

> **重要提示**：本项目中的所有代码都是可运行的。你可以逐步运行每一段代码，观察中间结果，深入理解策略的每个环节。不要直接跳到最终结果——理解过程比看到结果更重要。

### 1.8 学习建议

1. **动手实践**：从头到尾自己输入（而非复制粘贴）每一段代码
2. **修改参数**：尝试调整MA周期、ATR倍数、止损比例，观察结果变化
3. **添加标的**：在单只股票验证通过后，扩展到多只股票
4. **记录笔记**：将遇到的问题和解决方案记录下来
5. **延伸阅读**：完成本项目后，尝试将策略改为做空的版本，或添加更多因子
