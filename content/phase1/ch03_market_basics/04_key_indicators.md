## 4.1 成交量加权平均价格（VWAP）

VWAP（Volume-Weighted Average Price）是成交量加权平均价格，被机构投资者广泛用作交易执行质量的基准。VWAP 将价格按其对应的成交量赋予不同权重，使得高成交量时段的价格对平均值的影响更大。

### 公式

```
VWAP = Σ(Priceᵢ × Volumeᵢ) / Σ(Volumeᵢ)
```

其中 `Priceᵢ` 通常取每根K线的典型价格（Typical Price）：

```
典型价格 = (High + Low + Close) / 3
```

### VWAP 的实际意义

- **执行基准**：如果买入均价低于当日VWAP，说明执行质量优秀（"跑赢VWAP"）；反之则执行不佳。这是基金公司考核交易员的核心指标之一。
- **支撑/阻力**：VWAP常被日内交易员视为动态支撑或阻力线。价格在VWAP上方运行时偏向强势，跌破VWAP则可能转弱。
- **算法交易**：TWAP（时间加权）和 VWAP 算法是两种最基础的拆单执行算法，前者按时间均匀分配，后者按历史成交量分布分配。

```python
# VWAP 计算示例
def calculate_vwap(df):
    typical_price = (df['high'] + df['low'] + df['close']) / 3
    vwap = (typical_price * df['volume']).cumsum() / df['volume'].cumsum()
    return vwap
```

> VWAP 是一个**日内指标**，每天重新初始化。隔夜不累积计算，这与移动均线等趋势指标的计算方式不同。

## 4.2 买卖价差（Bid-Ask Spread）

买卖价差是市场上最佳买入报价和最佳卖出报价之间的差额，也是衡量市场质量的直接指标。

```
买卖价差 = 最低卖价(Ask) - 最高买价(Bid)
相对价差 = (Ask - Bid) / MidPrice × 100%
其中 MidPrice = (Ask + Bid) / 2
```

### 价差的经济含义

| 价差特征 | 市场状态 | 对策略的影响 |
|----------|---------|------------|
| 价差极小（< 0.1%） | 高流动性，深度足 | 交易成本低，适合高频 |
| 价差正常（0.1%-0.3%） | 流动性正常 | 普通算法交易可接受 |
| 价差偏大（0.3%-1%） | 流动性偏紧 | 需考虑冲击成本 |
| 价差巨大（> 1%） | 流动性枯竭 | 避免交易 |

买卖价差直接影响量化策略的交易成本。一只价格 10 元的股票，如果买卖价差为 0.02 元（相对价差 0.2%），意味着一买一卖的"摩擦成本"至少为 0.2%。高频策略中，频繁进出的累计成本不可忽视。

> **做市商视角**：做市商赚取的就是买卖价差——低价买入，高价卖出。但做市商需承担库存风险（持仓方向与市场走势相反的风险），因此价差的本质是对做市商承担风险的补偿。

## 4.3 市场流动性（Market Liquidity）

流动性（Liquidity）指在不显著影响价格的情况下，快速买入或卖出资产的能力。流动性是量化策略能否"理论变现实"的关键。

### 流动性的三个维度

```
┌──────────────────────────────────────────────────┐
│              市场流动性的三维度量                  │
├──────────────┬──────────────┬────────────────────┤
│   紧度       │    深度      │       弹性          │
│  (Tightness) │  (Depth)     │   (Resiliency)     │
├──────────────┼──────────────┼────────────────────┤
│ 买卖价差越小 │ 盘口挂单越多 │ 价格偏离后恢复越快  │
│ 市场越紧凑   │ 市场越有深度 │ 市场越有弹性        │
└──────────────┴──────────────┴────────────────────┘
```

### 常用的流动性指标

| 指标 | 计算方式 | 说明 |
|------|---------|------|
| 成交量 | 每日成交总股数 | 最基础的流量度量 |
| 换手率 | 成交量/流通股本 | 标准化后可跨股比较 |
| 买卖价差 | Ask - Bid | 紧度度量 |
| Amihud非流动性比率 | |日收益率| / 日成交额 | 价格冲击代理变量 |
| 盘口深度 | 各档挂单量之和 | 深度度量（需要Level 2数据）|

### Amihud 指标详解

Amihud 非流动性比率是最常用的学术研究指标：

```
ILLIQ = |Rᵢ| / (Pᵢ × Volumeᵢ)
```

其中 Rᵢ 是第 i 日的收益率。该指标衡量"单位成交额引起的价格变动"，数值越大说明股票流动性越差。研究表眀 Amihud 指标与未来股票收益率存在显著正相关性——低流动性股票往往有更高的预期收益（流动性溢价）。

## 4.4 波动率基础（Volatility）

波动率（Volatility）衡量资产价格在特定时期内的变化幅度，是量化交易中最重要的风险度量参数之一。

### 历史波动率（Historical Volatility）

历史波动率基于已发生的价格数据计算，反映过去的实际波动情况：

```
日收益率：rᵢ = ln(Pᵢ / Pᵢ₋₁)
日波动率：σ_daily = std(rᵢ)
年化波动率：σ_annual = σ_daily × √252
```

年化波动率 20% 意味着，在 68% 的置信区间（1个标准差）内，年收益率大约在 ±20% 范围内波动。注意：这是基于正态分布的简化假设，实际金融数据的"尖峰肥尾"特性意味着极端事件的发生概率远高于正态分布预测。

### 隐含波动率（Implied Volatility）

隐含波动率（IV）是从期权市场价格中反推的波动率预期：

```
期权价格 = f(S, K, T, r, σ)  通过Black-Scholes等模型反解 σ
```

隐含波动率反映的是**市场对未来波动的预期**，而非过去的历史数据。VIX 指数就是 S&P 500 期权的隐含波动率加权平均值，常被称为"恐慌指数"。

### 历史 vs 隐含波动率对比

| 维度 | 历史波动率 | 隐含波动率 |
|------|----------|----------|
| 数据来源 | 历史价格序列 | 期权市场价格 |
| 时间属性 | 回顾已发生 | 前瞻预期 |
| 获取难度 | 低（只需价格） | 需要期权数据 |
| 策略用途 | 风控参数设定 | 波动率交易、择时 |
| 反应速度 | 滞后于市场 | 即时反映预期 |

```python
# 历史波动率计算
import numpy as np

def historical_volatility(close_prices, window=20, trading_days=252):
    log_returns = np.log(close_prices / close_prices.shift(1))
    rolling_std = log_returns.rolling(window).std()
    annual_vol = rolling_std * np.sqrt(trading_days)
    return annual_vol
```

## 4.5 盘口深度（Market Depth）

盘口（Order Book / 盘口）记录了当前市场上所有未成交的限价单，分为买盘（Bid）和卖盘（Ask）两方。

### 盘口结构示意

```
        卖五 (Ask 5)  10.15  500手  ← 远端卖单
        卖四 (Ask 4)  10.13  300手
        卖三 (Ask 3)  10.12  800手
        卖二 (Ask 2)  10.11  400手
        卖一 (Ask 1)  10.10  600手  ← 最佳卖价
  ───────────────────────────────────── 买卖价差
        买一 (Bid 1)  10.08  700手  ← 最佳买价
        买二 (Bid 2)  10.07  500手
        买三 (Bid 3)  10.06 1000手
        买四 (Bid 4)  10.05  200手
        买五 (Bid 5)  10.03  900手  ← 远端买单
```

### 盘口不平衡（Order Book Imbalance）

盘口不平衡是高频交易中常用的信号：

```
OBI = (∑Bid_Volume - ∑Ask_Volume) / (∑Bid_Volume + ∑Ask_Volume)
```

OBI 为正表示买盘强于卖盘，短期内价格可能上涨；OBI 为负则反之。该指标对预测短期价格变动（秒级/分钟级）具有统计显著性。

| 指标 | 计算方式 | 策略应用 |
|------|---------|---------|
| 盘口深度 | 各档挂单量加权 | 判断大单是否可被吸收 |
| OBI | 买卖盘总量差/总量 | 短期价格预测 |
| 深度衰减 | 挂单量随价格层级递减速度 | 评估市场弹性 |
| 大单检测 | 单档挂单突增 | 识别机构行为迹象 |

> Level 2 数据包含完整的盘口快照，但数据量巨大（一只股票一天可产生数十GB数据），对存储和计算提出了很高要求。DolphinDB 的分布式架构正是为解决此类海量时序数据的存储与查询而设计的。
