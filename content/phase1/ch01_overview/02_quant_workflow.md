## 2.1 量化交易的完整流程

量化交易不是一个简单的"写个程序自动买卖"，而是一个完整的系统工程。标准的量化交易工作流程包含五个核心阶段，形成一个闭环：

```
数据获取 → 策略研发 → 回测验证 → 执行交易 → 绩效监控
    ↑                                                ↓
    └────────────────── 反馈优化 ←──────────────────┘
```

这个流水线（Pipeline）的每个环节都至关重要，任何一个环节的薄弱都可能导致整个策略失败。

## 2.2 第一阶段：数据获取与管理

数据是量化交易的"原材料"。没有高质量的数据，再精妙的模型也无济于事。

### 数据类型分类

| 类别 | 具体数据 | 来源示例 | 频率 |
|------|----------|----------|------|
| 行情数据 | OHLCV、Tick数据、订单簿 | 交易所、数据商（Wind、Choice） | Tick ~ 日频 |
| 基本面数据 | 财报、估值指标、行业分类 | 上市公司公告、Wind、东方财富 | 季频/年频 |
| 另类数据 | 卫星图像、社交媒体情绪 | 第三方数据供应商 | 实时 ~ 日频 |
| 宏观经济数据 | GDP、CPI、货币供应量 | 国家统计局、央行 | 月频 ~ 年频 |
| 另类数据 | 供应链信息、信用卡消费数据 | 第三方数据商 | 日频 ~ 月频 |

### 数据质量保证

> **"Garbage in, garbage out"**（垃圾进，垃圾出）是量化交易的第一铁律。数据质量直接决定策略的可靠性。以下问题必须警惕：

- **幸存者偏差（Survivorship Bias）**：历史数据库只包含"存活至今"的股票，忽略了已退市的股票，会导致回测收益被严重高估
- **前视偏差（Look-Ahead Bias）**：使用了当时不可知的信息。例如，用5月1日的年报数据回测4月30日的交易，而年报实际上是4月30日收盘后才发布的
- **除权除息处理**：如果不进行复权处理，分红派息会导致股价数据出现断点
- **停牌处理**：停牌期间的数据如何填充，会显著影响策略信号

```python
# Python示例：检查前视偏差
import pandas as pd

# 年报公告日是4月30日，但财务数据实际在5月1日才能获取
report_date = pd.Timestamp('2024-04-30')
available_date = pd.Timestamp('2024-05-01')

# 错误做法：在4月30日使用年报数据做交易信号
# 正确做法：只能从5月1日开始使用这份年报数据
```

## 2.3 第二阶段：策略研发

策略研发是量化交易中最具创造性的环节。这个过程将投资思想转化为可执行的数学模型和代码。

### 策略研发的核心步骤

**1. 投资思想（Idea Generation）**
策略的思想来源非常广泛：
- 经典金融文献中的因子（Fama-French 三因子、动量因子等）
- 市场微观结构观察（bid-ask价差模式、订单簿不平衡）
- 行为金融学理论（投资者过度反应、锚定效应）
- 数据挖掘发现的新规律

**2. 信号生成（Signal Generation）**
将投资思想量化为具体的交易信号。信号通常是一个数值，正值代表做多，负值代表做空。信号的强度（绝对值大小）代表仓位的大小。

```python
# 简单示例：基于移动平均线的信号生成
def generate_ma_signal(close_prices):
    """
    生成移动平均线交叉信号
    正值：短均线上穿长均线，看多
    负值：短均线下穿长均线，看空
    """
    ma_short = close_prices.rolling(window=20).mean()
    ma_long = close_prices.rolling(window=60).mean()
    
    # 信号 = 短均线 - 长均线（正值=多头，负值=空头）
    signal = ma_short - ma_long
    return signal
```

**3. 仓位管理（Position Sizing）**
根据信号的强度和当前的风险水平，确定应该持有多少仓位。常用方法：
- **等权重**：每个信号分配相同资金
- **波动率倒数加权**：波动大的品种给较低仓位
- **凯利公式（Kelly Criterion）**：根据胜率和赔率计算最优仓位
- **风险平价（Risk Parity）**：使各品种对组合风险的贡献相等

**4. 组合构建（Portfolio Construction）**
将多个信号综合为最终的投资组合，需要考虑：
- 行业/板块分散化
- 多空平衡（净敞口控制）
- 换手率约束（控制交易成本）

### 一个完整的策略示例

```python
import pandas as pd
import numpy as np

def momentum_strategy(df, lookback=60, top_n=20):
    """
    简单的横截面动量策略：
    1. 计算过去N天的收益率排名
    2. 做多排名最高的top_n只股票
    3. 做空排名最低的top_n只股票
    """
    # 计算过去lookback天的累计收益率
    returns = df['close'].pct_change(lookback)
    
    # 按收益率排名，分位数越大越好
    rank = returns.rank(pct=True)
    
    # 前top_n%看多，后top_n%看空
    long_signal = (rank >= 1 - top_n/100).astype(int)
    short_signal = (rank <= top_n/100).astype(int) * -1
    
    return long_signal + short_signal
```

> **关键原则**：在策略研发阶段，务必保持"样本外数据"（Out-of-Sample Data）的独立性。不要反复使用同一批数据进行参数优化后再测试，这会导致严重的过拟合。

## 2.4 第三阶段：回测验证

回测（Backtesting）是策略研发的"试金石"。通过模拟策略在历史数据上的表现，评估策略的有效性和稳健性。

### 回测的核心评估指标

| 指标 | 计算公式 | 说明 |
|------|----------|------|
| 年化收益率 | (终值/初值)^(252/天数) - 1 | 折合到每年的平均收益率 |
| 年化波动率 | 日收益率标准差 × √252 | 衡量收益的不稳定性 |
| 夏普比率 | (年化收益 - 无风险利率) / 年化波动率 | 风险调整后的收益，>1为优秀 |
| 最大回撤 | (峰值 - 谷值) / 峰值 | 历史最大亏损幅度 |
| 卡玛比率 | 年化收益 / 最大回撤 | 衡量单位回撤带来的收益 |
| 胜率 | 盈利交易次数 / 总交易次数 | 交易成功比例 |
| 盈亏比 | 平均盈利 / 平均亏损 | 盈利交易的平均收益与亏损交易平均亏损之比 |

### 回测中的常见偏误

1. **过拟合（Overfitting）**：参数过度优化，导致样本内表现优异但样本外表现糟糕
2. **未来函数**：使用了当时不存在的未来信息（如用当日最高价计算买入信号）
3. **交易成本忽略**：未计入手续费、滑点、冲击成本
4. **幸存者偏差**：回测数据只包含现存股票

```python
# Python：计算核心回测指标
def calculate_metrics(daily_returns, risk_free_rate=0.03):
    """计算策略的核心绩效指标"""
    ann_return = (1 + daily_returns.mean()) ** 252 - 1
    ann_volatility = daily_returns.std() * np.sqrt(252)
    sharpe = (ann_return - risk_free_rate) / ann_volatility
    
    cum_returns = (1 + daily_returns).cumprod()
    running_max = cum_returns.expanding().max()
    drawdown = (cum_returns - running_max) / running_max
    max_drawdown = drawdown.min()
    calmar = ann_return / abs(max_drawdown)
    
    return {
        '年化收益率': f'{ann_return:.2%}',
        '年化波动率': f'{ann_volatility:.2%}',
        '夏普比率': f'{sharpe:.2f}',
        '最大回撤': f'{max_drawdown:.2%}',
        '卡玛比率': f'{calmar:.2f}'
    }
```

## 2.5 第四阶段：执行交易

策略通过回测验证后，就进入实盘执行阶段。执行环节的核心挑战是：

### 两大核心问题

**1. 交易成本控制**
- **佣金（Commission）**：券商收取的固定或按比例费用
- **印花税**：政府对交易的征税（A股卖出方征收0.1%）
- **滑点（Slippage）**：预期价格与实际成交价格之间的差异
- **市场冲击成本（Market Impact）**：大额订单对市场价格的负面影响

**2. 执行算法（Execution Algorithms）**
为降低冲击成本，大订单通常拆分为小订单逐步执行：
- **TWAP（时间加权平均价格）**：将订单均匀分配到一段时间
- **VWAP（成交量加权平均价格）**：根据历史成交量分布拆分订单
- **Implementation Shortfall**：平衡执行速度与市场冲击

### 交易系统的技术要求

```
┌─────────────────────────────────────────┐
│              交易系统架构                  │
├──────────┬──────────┬───────────────────┤
│  行情接入  │ 策略引擎  │    订单管理        │
│ (低延迟)  │ (信号生成) │ (OMS/EMS)        │
├──────────┼──────────┼───────────────────┤
│ 实时数据库 │ 风控模块  │  报告与日志系统    │
└──────────┴──────────┴───────────────────┘
```

## 2.6 第五阶段：绩效监控与迭代

实盘交易不是终点，而是新的起点。持续的监控和改进至关重要。

### 监控内容
- 策略是否按预期运行（信号是否正常生成）
- 交易成本是否在预期范围内
- 策略绩效是否出现显著偏离（可通过统计检验判断）
- 市场环境是否发生结构性变化

### 策略迭代循环

当策略绩效出现衰减迹象时，需要：
1. 判断是正常的回撤还是策略失效
2. 分析失效原因（市场结构变化、策略被复制、数据问题）
3. 进行策略改进或研发新策略
4. 重新进入回测——执行——监控的循环

> **终极警示**：量化交易最危险的时刻，不是策略出现大幅回撤的时候，而是当你因为连续盈利而放松警惕、加大杠杆的时刻。永远记住——历史不代表未来，统计规律不是物理定律。
