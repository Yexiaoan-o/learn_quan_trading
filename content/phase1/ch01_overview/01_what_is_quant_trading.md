## 1.1 量化交易的定义

量化交易（Quantitative Trading）是指利用数学模型、统计分析和计算机程序来制定和执行交易策略的投资方法。与传统的主观交易不同，量化交易将投资决策过程标准化、系统化和自动化。

量化交易的核心特征包括：

| 特征 | 说明 |
|------|------|
| **数据驱动** | 所有决策基于历史数据和实时数据的定量分析，而非主观判断 |
| **模型化** | 使用数学模型描述市场规律和交易逻辑 |
| **系统化** | 策略研发、回测、执行全流程由计算机系统完成 |
| **纪律性** | 严格执行既定策略，避免情绪干扰 |

> **核心理念**：量化交易的本质不是追求"预测市场"，而是通过概率优势和系统化执行，在大量交易中获取统计意义上显著的正期望收益。

### 量化交易的数学模型基础

量化交易的数学基础可以追溯到概率论与数理统计。假设我们有一个交易策略，每次交易盈利的概率为 p，亏损的概率为 1-p，平均盈利为 W，平均亏损为 L，则策略的期望收益为：

```
E = p × W - (1-p) × L
```

量化交易的目标是找到期望收益为正（E > 0）的策略，并通过大量交易使实际收益趋近于期望值（大数定律）。

例如，一个胜率仅为 40% 的策略，如果每次盈利平均赚 300 元，每次亏损平均亏 100 元：

```
E = 0.4 × 300 - 0.6 × 100 = 120 - 60 = 60
```

即使胜率低于 50%，该策略依然具有正的期望收益。这就是量化交易中常说的"截断亏损，让利润奔跑"（Cut Losses Short, Let Profits Run）的数学原理。

## 1.2 量化交易的发展历史

量化交易的发展经历了几个重要阶段：

### 萌芽期（1950s-1970s）
- **1952年**：Harry Markowitz 提出现代投资组合理论（MPT），用均值-方差模型量化风险与收益
- **1964年**：William Sharpe 提出资本资产定价模型（CAPM），建立单因子定价框架
- **1973年**：Black-Scholes 期权定价模型问世，为衍生品定价提供数学工具

### 成长期（1980s-1990s）
- **1983年**：芝加哥商品交易所（CME）推出标普500指数期货
- **1988年**：Renaissance Technologies 成立大奖章基金（Medallion Fund），年化收益超66%
- **1993年**：Fama-French 三因子模型发表，系统化解释股票横截面收益
- **1998年**：Long-Term Capital Management (LTCM) 危机，凸显量化模型风险

### 成熟期（2000s至今）
- **2007年**：Two Sigma 成立，成为量化对冲基金新锐
- **2014年**：全球量化基金资产规模突破 1 万亿美元
- **2020年代**：机器学习、深度学习大规模应用于量化交易
- **2023年**：大型语言模型（LLM）和生成式AI开始辅助策略研发

> **里程碑事件**：Renaissance Technologies 的 Medallion Fund 自1988年至2018年的30年间，扣除费用前的年化收益率高达 66%，被认为是历史上最成功的量化基金。其创始人 Jim Simons 是顶尖数学家，团队大量招募数学、物理、计算机科学的博士，而非传统金融背景人才，这深刻改变了华尔街的人才结构。

## 1.3 传统交易 vs 量化交易

### 决策方式对比

```python
# 传统交易者的决策过程（主观判断）
"""
交易者看到股价突破20日均线，结合成交量的判断，
以及对市场消息的理解，主观决定买入1000股。
这个决策依赖于交易者的经验、情绪和直觉。
"""

# 量化交易者的决策过程（系统化）
def should_buy(close_price, ma_20, volume, volume_ma_20):
    # 条件1：价格突破20日均线
    condition1 = close_price > ma_20
    
    # 条件2：成交量放大（超过20日均量的1.5倍）
    condition2 = volume > volume_ma_20 * 1.5
    
    # 条件3：价格不能偏离均线超过5%（防止追高）
    condition3 = close_price / ma_20 < 1.05
    
    return condition1 and condition2 and condition3
```

### 全方位对比

| 维度 | 传统交易 | 量化交易 |
|------|----------|----------|
| **决策依据** | 经验、直觉、消息 | 数据、模型、统计 |
| **信息处理** | 关注几只到几十只股票 | 可同时处理数千只股票 |
| **情绪影响** | 容易受贪婪和恐惧影响 | 严格执行，无情绪波动 |
| **可复制性** | 难以复制，依赖个人能力 | 策略可标准化复制 |
| **回测能力** | 无法精确回测 | 可对历史数据进行严谨回测 |
| **交易频率** | 日频为主 | 日频到高频（微秒级） |
| **优势来源** | 信息优势和研判能力 | 速度优势和统计优势 |

### 量化交易适合的场景

- 处理海量数据，寻找统计规律
- 高频或大容量交易，人工无法执行
- 跨市场、多品种的套利策略
- 需要严格风控和纪律执行的场景

量化交易**不适合**的场景：
- 极端事件驱动（如突发的重大政策变化）
- 流动性极差、数据不完整的小众市场
- 依赖深度产业理解和商业判断的长期价值投资

## 1.4 量化交易的收益来源

量化策略的收益通常来源于以下几个方面：

### 风险溢价（Risk Premium）
市场对承担风险的补偿。例如：
- **股票风险溢价**：承担股市波动风险，获取相对于无风险利率的超额回报
- **流动性溢价**：持有流动性较差资产所获得的额外补偿
- **信用溢价**：承担违约风险所获取的额外收益

### 市场非有效性（Market Inefficiency）
- **行为偏差**：投资者系统性犯错误（追涨杀跌、过度反应、羊群效应）
- **信息传递延迟**：信息在市场中并非瞬间被所有投资者消化
- **制度性约束**：某些机构投资者因制度限制必须买卖，产生定价偏差

### 技术优势
- **执行速度**：更快的行情获取和订单执行
- **数据处理**：处理非结构化数据（新闻、社交媒体）的能力
- **算法复杂度**：能发现传统分析无法发现的复杂模式

> **重要认知**：量化交易并非"无风险套利"。任何能稳定产生超额收益的策略，其本质要么是承担了某种系统性风险，要么是利用了市场的暂时非有效性。随着市场参与者增多，非有效性会逐渐被套利行为消除，超额收益会衰减——这就是策略的"alpha衰减"现象。优秀的量化交易者需要持续研发新策略，保持竞争优势。
