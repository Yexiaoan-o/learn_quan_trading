## 回测的基本概念

### 什么是回测？

回测（Backtesting）是量化交易中最为核心的环节之一。简单来说，回测就是利用历史市场数据，模拟交易策略在过去某段时间内的表现，从而评估策略是否具有盈利能力的过程。回测的本质是回答一个根本性问题：**如果我在过去按照这套规则进行交易，结果会怎样？**

在量化交易中，任何一个策略在被投入实盘之前，都必须经过严格的回测检验。这不仅仅是为了验证策略能否赚钱，更重要的是理解策略的风险特征、发现潜在的逻辑缺陷、以及优化策略参数。回测是整个量化交易流程中连接"理论"与"实践"的桥梁——它将你在纸上或代码中构思的交易逻辑转化为可量化的绩效指标，让你能够以数据驱动的方式做出决策。

回测的核心流程包括以下几个步骤：

1. **数据准备**：获取并清洗历史行情数据（价格、成交量等）以及可能需要的其他辅助数据（财务数据、市场数据等）
2. **策略定义**：将交易逻辑转化为明确的、可执行的规则，包括买入条件、卖出条件、仓位管理规则
3. **模拟执行**：按照时间顺序（或向量化方式）遍历历史数据，在每个时间点判断策略是否触发交易信号
4. **性能评估**：计算并分析策略的各项绩效指标（收益率、最大回撤、夏普比率等）
5. **分析与优化**：根据回测结果分析策略的优缺点，进行参数优化或逻辑改进

下面的简单示例展示了回测的基本逻辑：

```python
import pandas as pd
import numpy as np

# 模拟历史价格数据
np.random.seed(42)
dates = pd.date_range('2020-01-01', '2022-12-31', freq='B')
prices = 100 * (1 + np.random.randn(len(dates)).cumsum() * 0.01)
df = pd.DataFrame({'close': prices}, index=dates)

# 计算移动平均线
df['ma_short'] = df['close'].rolling(20).mean()
df['ma_long'] = df['close'].rolling(60).mean()

# 生成交易信号
df['signal'] = 0
df.loc[df['ma_short'] > df['ma_long'], 'signal'] = 1   # 买入/持有
df.loc[df['ma_short'] <= df['ma_long'], 'signal'] = 0  # 卖出/空仓

# 计算每日收益率
df['returns'] = df['close'].pct_change()
df['strategy_returns'] = df['signal'].shift(1) * df['returns']

# 计算累计收益
df['cumulative_returns'] = (1 + df['strategy_returns']).cumprod()

print(f"总收益率: {(df['cumulative_returns'].iloc[-1] - 1) * 100:.2f}%")
print(f"年化收益率: {((df['cumulative_returns'].iloc[-1]) ** (252/len(df)) - 1) * 100:.2f}%")
```

### 为什么回测如此重要？

回测的重要性体现在以下多个层面：

**1. 策略可行性验证**
每一套交易策略本质上都是一种"假设"——假设某种市场规律存在并且能够被捕捉。回测是将这些假设付诸检验的过程。如果策略在历史数据上都表现不佳，那么几乎没有理由相信它在未来会有好的表现。回测为你提供了策略是否值得继续深入研究的初步判断依据。

**2. 风险认知**
回测不仅能告诉你策略能赚多少钱，更重要的可能是告诉你策略会亏多少钱。最大回撤、波动率、胜率等指标能够帮助你理解策略在不同市场环境下的风险特征。很多新手交易者过于关注收益而忽视风险，而回测提供的风险指标能够帮助建立正确的风险认知。

**3. 参数敏感性分析**
大多数策略都包含可调参数（例如均线的周期、RSI的阈值等）。回测可以帮助你理解策略对这些参数变化的敏感程度：参数的小幅调整是否会导致策略绩效的剧烈变化？这能帮助你判断策略的稳健性，避免过度优化的陷阱。

**4. 交易成本评估**
在实际交易中，手续费、滑点、印花税等交易成本不可忽视。回测中可以方便地引入这些成本因素，评估它们在多大程度上侵蚀策略利润。很多在"零成本"假设下表现优异的策略，加入现实交易成本后可能变得不再盈利。

**5. 多策略对比**
当你有多个备选策略时，回测提供了一套统一的评估框架，让你能够在相同的市场环境和时间范围内横向对比不同策略的表现，做出更有依据的决策。

**6. 心理准备**
通过回测，你可以提前了解策略最糟糕的时候可能是什么样子——连续亏损的次数、最大回撤的幅度。这有助于在实盘中面对不利行情时保持冷静，坚守纪律，避免因情绪化决策而提前终止有效的策略。

### 回测中的常见陷阱

回测虽然强大，但若使用不当，会产生极具误导性的结果。以下是回测中最常见的几个陷阱：

#### 1. 前视偏差（Look-Ahead Bias）

> **前视偏差是回测中最常见也最致命的错误**——在某个时间点使用了该时间点之后才能获得的信息。这会导致回测结果被人为美化，策略在实盘中往往表现远逊于预期。

**常见的前视偏差场景**：

| 场景 | 错误做法 | 正确做法 |
|------|---------|---------|
| 使用财务数据 | 在财报公告日之前使用当季财务数据 | 确保只在财务数据实际发布后才使用 |
| 数据计算 | 使用全样本数据计算均值/标准差 | 使用滚动窗口或扩展窗口（只用当前时间点之前的数据） |
| 时间对齐 | 使用当日收盘价判断买入并在当日买入 | 使用前一日收盘价判断，次日开盘价执行 |
| 股票池筛选 | 使用未来业绩筛选股票 | 使用当前时间点已知的信息筛选 |
| 幸存者偏差 | 回测只包含"活到现在"的股票 | 应包含已退市的股票 |

以下是一个前视偏差的代码示例及其修正：

```python
import pandas as pd
import numpy as np

# 模拟数据：包含"未来信息"的版本
dates = pd.date_range('2020-01-01', '2022-12-31')
n = len(dates)

# 错误示例：使用全样本的均值和标准差进行标准化
df = pd.DataFrame({
    'close': 100 * (1 + np.random.randn(n).cumsum() * 0.01),
    'volume': np.random.randint(1000000, 10000000, n)
}, index=dates)

# 错误做法：用全样本均值——这引入了前视偏差！
# df['zscore_full'] = (df['close'] - df['close'].mean()) / df['close'].std()

# 正确做法：使用滚动窗口（只用历史数据）
df['zscore_rolling'] = (df['close'] - df['close'].rolling(252).mean()) / df['close'].rolling(252).std()

print("前60天的滚动Z-score（前60天为NaN因为需要预热期）：")
print(df['zscore_rolling'].iloc[252:260])
```

#### 2. 幸存者偏差（Survivorship Bias）

> **幸存者偏差**是指回测中只包含了"存活至今"的股票，而忽视了那些已经退市、被收购或破产的公司。这会导致回测结果被高估，因为表现差的股票已经消失了。

幸存者偏差尤其在对小盘股、成长股进行回测时严重。那些最终倒闭的公司通常在倒闭前经历了持续下跌，排除它们意味着你过度美化了过去可能面临的风险。

**如何避免幸存者偏差**：

1. 使用包含历史成分股变更信息的数据库（如CRSP、Compustat等专业数据库）
2. 确保数据集中包含已退市股票在退市前的全部历史数据
3. 在回测中正确处理退市事件（按退市价格结算持仓）
4. 使用"无幸存者偏差"的指数成分股历史记录

```python
# 模拟处理退市股票的简单示例
import pandas as pd
import numpy as np

def handle_delisting(prices_df, delisting_date, delisting_price=None):
    """
    prices_df: 股票价格DataFrame，index为日期
    delisting_date: 退市日期
    delisting_price: 退市时结算价格（如果有），否则使用最后交易价格
    """
    if delisting_date in prices_df.index:
        if delisting_price is not None:
            prices_df.loc[delisting_date, 'close'] = delisting_price
        # 退市后不再交易，后续收益率应为NaN而非0
        prices_df.loc[prices_df.index > delisting_date, 'close'] = np.nan
    return prices_df
```

#### 3. 过度优化（Overfitting / Curve-Fitting）

过度优化是指通过不断调整策略参数，使得策略在历史数据上表现极其出色，但实际上只是在"记住"了历史噪声，在未来的新数据上表现不佳。

**过度优化的典型表现**：
- 回测收益极高但样本外测试收益极低
- 参数越多、回测收益越高（参数增长带来的收益改善是非线性的）
- 策略对参数变化极其敏感
- 策略逻辑过于复杂，没有合理的经济学原理支撑

**如何避免过度优化**：

1. **样本外测试**：将历史数据分为"训练集"（用于策略开发和参数选择）和"测试集"（用于最终验证，仅在开发完成后使用一次）
2. **交叉验证**：使用多段不同的样本外时间段进行验证
3. **简化策略**：越简单的策略通常越稳健，优先考虑逻辑清晰、参数少的策略
4. **敏感性分析**：检查参数微小变化是否会导致绩效剧烈波动
5. **保持怀疑**：如果夏普比率超过3甚至5，很可能存在问题

```python
# 样本内/样本外分割示例
import pandas as pd

# 假设df包含完整的历史数据
df = pd.DataFrame(...)  # 你的历史数据
total_days = len(df)
split_idx = int(total_days * 0.7)  # 70%样本内，30%样本外

df_in_sample = df.iloc[:split_idx]     # 样本内：用于策略开发
df_out_sample = df.iloc[split_idx:]    # 样本外：用于最终验证

print(f"样本内时间范围: {df_in_sample.index[0]} 至 {df_in_sample.index[-1]}")
print(f"样本外时间范围: {df_out_sample.index[0]} 至 {df_out_sample.index[-1]}")
```

#### 4. 数据窥探偏差（Data-Snooping Bias）

数据窥探是指研究人员在开发过程中多次使用同一组数据来测试和改进策略，每次看到回测结果后再对策略进行微调。这样反复"偷看"数据的过程，实际上是在让策略逐渐"适应"这组特定历史数据的特性，最终导致过拟合。

避免数据窥探的核心原则是：**在策略开发的迭代过程中，严格保护测试集数据不被使用**。最好的做法是按照以下流程进行：

1. 先在训练集上完成所有策略开发、参数优化
2. 一旦策略最终确定，在测试集上运行**仅一次**回测
3. 以测试集的结果作为策略真实表现的估计
4. **绝对不要**因为测试集结果不好而回去修改策略再测

#### 5. 交易成本忽视

很多回测框架默认忽略了交易成本，这在现实中是不切实际的。交易成本包括：

- **佣金/手续费**：每次交易支付给券商的费用
- **印花税**：部分市场（如A股）在卖出时收取的税费
- **滑点（Slippage）**：期望成交价与实际成交价之间的差额
- **冲击成本（Market Impact）**：大额订单本身推动价格向不利方向移动造成的成本

对于高频策略和资金规模较大的策略，交易成本的影响尤为显著。建议在回测中始终加入合理的交易成本假设。

```python
def apply_transaction_cost(price, shares, cost_rate=0.0003, slippage=0.0001):
    """
    计算实际成交成本
    cost_rate: 手续费率（如万分之三）
    slippage: 滑点比例（如万分之一）
    """
    trade_value = price * shares
    commission = trade_value * cost_rate
    slippage_cost = trade_value * slippage
    # A股卖出时需缴纳印花税（0.1%）
    stamp_duty = trade_value * 0.001 if shares < 0 else 0  # shares < 0 表示卖出
    total_cost = commission + slippage_cost + stamp_duty
    return total_cost
```

### 回测的基本原则

要获得可靠的回测结果，应当遵循以下基本原则：

1. **使用高质量数据**：数据质量决定了回测结果的上限。脏数据、错误数据会导致不可靠的结论
2. **保持时间逻辑**：确保策略在任何时间点只能使用该时间点之前可获得的信息
3. **考虑交易成本**：真实世界中有各种摩擦成本，回测中必须包含
4. **防止过拟合**：使用样本外测试、交叉验证等方法
5. **保持策略简单**：简单策略通常比复杂策略更稳健
6. **关注风险指标**：不要只看收益，也要关注最大回撤、夏普比率等风险指标
7. **理解市场机制**：策略应该在市场中切实可行（流动性、交易时间、涨跌停限制等）

### 总结

回测是量化交易策略开发的基石。正确地进行回测，能够为交易决策提供坚实的数据支撑；而错误的回测方法，则会导致对策略能力的过度乐观估计，最终在实盘中遭受不应有的损失。在后续的章节中，我们将逐步学习回测的各种具体技术——包括如何进行向量化回测、如何构建事件驱动回测引擎、以及如何计算和分析回测的各项绩效指标。
