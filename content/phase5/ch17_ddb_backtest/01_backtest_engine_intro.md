## DolphinDB回测引擎概述

### 1.1 什么是回测引擎

回测引擎（Backtest Engine）是量化交易策略开发中不可或缺的工具。它能够在历史数据上模拟执行交易策略，评估策略在过去的理论表现，帮助研究者在实盘前发现策略的优缺点。

DolphinDB的回测引擎是一个基于事件驱动架构的高性能回测框架，内置在DolphinDB服务器中，无需额外安装第三方库。它能够充分利用DolphinDB的分布式计算能力，在大量标的上高效运行回测。

> **事件驱动**：回测引擎按时间顺序"播放"历史行情数据，当新的行情到来时（Bar事件或Tick事件）触发策略逻辑，模拟真实交易环境下的决策过程。

### 1.2 核心特性

DolphinDB回测引擎具备以下关键特性：

| 特性 | 说明 | 价值 |
|------|------|------|
| **事件驱动** | 基于Bar/Tick事件触发策略逻辑 | 贴近真实交易流程 |
| **模拟撮合** | 内置订单撮合引擎，支持多种订单类型 | 准确模拟成交情况 |
| **性能分析** | 自动生成收益曲线、回撤、夏普比等指标 | 快速评估策略质量 |
| **分布式计算** | 支持在多标的上并行回测 | 大幅度提升回测效率 |
| **SQL集成** | 直接使用DolphinDB SQL获取和预处理数据 | 数据流无缝衔接 |

### 1.3 架构组件

回测引擎由以下核心组件构成：

#### 1.3.1 市场模拟器（Market Simulator）

市场模拟器负责按时间顺序"重放"历史行情数据。它从数据库读取指定时间范围内、指定标的的OHLCV数据，按时间顺序逐条推送给策略处理函数。

当策略发出订单时，市场模拟器根据下一个Bar或Tick的行情数据来模拟撮合，判断订单是否成交、成交价格和成交量。

#### 1.3.2 订单管理器（Order Manager）

订单管理器负责维护所有未成交订单的生命周期：

- **订单接收**：从策略接收买卖指令，生成订单对象
- **订单状态**：跟踪订单状态变化（待成交 → 部分成交 → 完全成交 / 已撤销）
- **撮合检查**：每当新行情到来时，检查限价单是否满足成交条件
- **订单记录**：记录每笔订单的完整信息用于事后分析

```python
#                  \-> CANCELLED
```

#### 1.3.3 持仓追踪器（Position Tracker）

持仓追踪器实时记录和管理账户的持仓信息、资金变动和交易记录：

- 当前持仓数量、成本价和浮动盈亏
- 可用资金和总资产
- 历史交易记录列表
- 每日净值序列

### 1.4 与Python回测框架对比

量化交易领域最常用的Python回测框架包括backtrader和zipline（Quantopian）。将DolphinDB回测引擎与它们对比：

| 对比维度 | DolphinDB | backtrader | zipline |
|----------|-----------|------------|---------|
| **运行环境** | DolphinDB服务器 | Python本地 | Python本地 |
| **性能** | 高（分布式、C++内核） | 中 | 中 |
| **多标的并行** | 原生支持 | 需自行实现 | 支持Pipeline |
| **数据管理** | 集成数据库 | 外部数据源 | 自定义Bundle |
| **学习曲线** | 中等 | 较低 | 中等 |
| **社区生态** | 较小 | 丰富 | 已停止维护 |
| **适用场景** | 大规模多标的回测 | 单/少量标的回测 | 基本面因子回测 |

**DolphinDB回测引擎的优势**：
1. **性能卓越**：C++底层实现，支持分布式并行回测，处理上千只股票的数据远比Python快
2. **数据一体**：数据存储、因子计算、回测分析在同一平台完成，避免数据在不同系统间搬运
3. **生产就绪**：回测代码可以快速转化为实盘代码，减少"回测与实盘不一致"的问题

**Python回测框架的优势**：
1. **生态丰富**：可结合pandas、matplotlib、sklearn等库，分析手段多样
2. **上手容易**：Python语法简单，文档和教程众多
3. **灵活性强**：可以轻松实现复杂的自定义逻辑

### 1.5 回测工作流程

典型的回测工作流程如下：

```
1. 数据准备 → 将历史行情数据导入DolphinDB
2. 策略编写 → 定义onBar/onTick处理函数
3. 配置参数 → 设置回测起止时间、初始资金、手续费等
4. 执行回测 → 调用回测引擎运行策略
5. 结果分析 → 查看收益曲线、回撤、夏普比等指标
6. 参数优化 → 调整策略参数，重复步骤3-5
```

> **最佳实践**：在Python中先完成策略的逻辑探索和初步测试（效率更高），确定策略框架后再用DolphinDB做大规模多标的回测（性能更优），两者结合实现效率最大化。

### 1.6 适用场景

DolphinDB回测引擎特别适合以下场景：

- **多标的批量回测**：同时对几百只股票运行同一策略
- **高频数据回测**：在分钟级甚至Tick级数据上回测（Python难以承受）
- **复杂因子策略**：策略依赖大量DolphinDB计算因子（避免数据导出）
- **生产级策略验证**：需要与实盘代码保持一致的场景

对于简单的单股策略初步验证，使用Python + pandas仍然是更高效的选择。
